# PRD 生成工作流

本技能将引导您通过严格的 5 阶段流程，生成高质量的 PRD 文档和原型。

## 阶段 1：业务调研与引导

**目标**：在动笔前收集所有必要的上下文信息。

1.  **阅读写作要求**：严格遵守 `references/prd_writing_requirements.txt`。
2.  **询问核心要素**：
    - **背景**：需求来源？目标是什么？成功指标？
    - **用户**：目标用户是谁？使用场景？
    - **约束**：技术限制？数据依赖？截止日期？
    - **页面类型**：涉及哪些类型的页面？(登录页 / 表单页 / 内容页 / 弹窗 / 其他)
    - **前端框架**：默认使用 **Arco Design**。确认是否有其他强制要求。

*(输出：一份结构化的问卷供用户回答)*

## 阶段 2：需求验证

**目标**：确保收集到的信息足以填充模版。

检查用户的回答是否覆盖了 `references/prd_template_2026.txt` 所需的所有章节：
- [ ] 清晰的背景和目标
- [ ] 定义了页面结构和类型
- [ ] 每种页面类型的交互规则
- [ ] 具体的业务逻辑（输入、输出、异常情况）

*如果信息缺失，请追问。在验证通过前，不要开始撰写 PRD。*

## 阶段 3：PRD 撰写 (核心)

**目标**：严格按照模版结构生成 Markdown 格式的 PRD。

**输入**：验证后的需求 + `references/prd_template_2026.txt`
**输出**：Markdown 格式的 PRD 文档。

**结构 (严格遵循 0210 模版)**：
1.  **需求背景**
2.  **前端框架参考** (Arco Design 规范)
3.  **云平台页面结构** (引用模版中的 4 类页面定义)
4.  **交互原则 (动态选择)**
    *仅包含用户实际涉及的页面类型：*
    - **4.1 登录页** (如果涉及登录)
    - **4.2 表单页** (如果涉及表单/管理)
    - **4.3 内容页** (如果涉及详情/只读)
    - **4.4 弹窗** (如果涉及弹窗操作)
5.  **页面层级** (层级说明/目录规范/工作流)
6.  **功能详细定义** (具体的功能特性规范)

## 阶段 4：原型生成

**目标**：通过 Vibe Coding 进行可视化验证。

**操作**：
1.  从生成的 PRD 中提取页面布局描述。
2.  **生成 HTML/CSS**：
    - 使用 **Arco Design** 风格。
    - 实现布局规范："固定顶栏"、"固定侧边栏"、"内容区自适应"。
    - **交互元素**：添加简单的点击事件（如切换侧边栏、打开弹窗、切换 Tab）以演示流程。
3.  保存为单文件 HTML 工件（例如 `prototype_form_page.html`）。

## 阶段 5：评分 (占位)

*目前为占位符。*
执行基础格式检查：
- 是否包含所有 6 个章节？
- 复杂功能是否有"特殊说明"？
