# PRD 调研清单 (Research Checklist)

在撰写 PRD 之前，请明确以下要点。这能确保产出物符合业务目标和技术约束。

## 1. 背景与目标
- **来源**：谁提出了这个需求？（例如：业务团队、技术债务、用户反馈）
- **目标**：我们要解决什么问题？期望达到什么结果？
- **成功指标**：如何衡量成功？（例如：转化率提升、效率提高）

## 2. 目标用户与场景
- **用户**：谁是主要用户？（例如：管理员、终端用户、VIP）
- **场景**：他们在什么情境下使用此功能？（例如：每月报告、实时监控）

## 3. 页面类型与交互（对模版选择至关重要）
*请勾选所有适用的类型：*
- [ ] **登录/鉴权**：登录、密码重置、SSO。
- [ ] **表单/管理**：数据录入、增删改查、带筛选的列表。
- [ ] **内容/详情**：只读视图、仪表盘、复杂数据可视化。
- [ ] **弹窗/对话框**：快捷操作、确认框、覆盖当前页的简单输入。

## 4. 技术与设计约束
- **前端框架**：默认为 **Arco Design**。是否有其他要求？
- **数据依赖**：是否需要考虑外部 API 或遗留数据结构？
- **截止日期**：是否有硬性上线时间？是否有分阶段要求？

## 5. 特定功能逻辑
- **核心流程**：描述主要的正常操作路径。
- **异常情况**：是否有需要处理的特定错误状态或边界条件？
